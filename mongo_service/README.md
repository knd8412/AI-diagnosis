# DB Service

A lightweight FastAPI microservice providing a MongoDB-backed REST API for
the AI Diagnosis Assistant. It handles persistence for two collections:
X-ray analysis results and patient chat logs.

## Overview

This service is the single point of contact for all MongoDB reads and writes.
Other services (image-processing, UI) communicate with it over HTTP rather
than connecting to MongoDB directly.

Two collections are managed within the `xray_service` database:

- **`analysis_results`** — stores X-ray inference outputs from the image-processing service
- **`chat_logs`** — stores per-session conversation history, upserted by `session_id`

## API Endpoints

### Analysis Results

#### `POST /results`

Saves a new analysis result document. Called by the image-processing service
after inference completes.

**Request body (JSON):** any result dict — fields are not strictly validated.
Expected fields from the image-processing service:

```json
{
  "pathologies": { "Cardiomegaly": 0.874, "Atelectasis": 0.312 },
  "confident_results": { "Cardiomegaly": 0.874 },
  "top_cams": { "Cardiomegaly": [[...], ...] },
  "primary_embedding": [...],
  "scan_file": "a1b2c3d4.png",
  "source_filename": "chest.png"
}
```

**Response:**
```json
{ "id": "<MongoDB ObjectId string>" }
```

---

#### `GET /results/{result_id}`

Returns a full result document by its MongoDB ObjectId.

---

#### `GET /results/{result_id}/pathologies`
#### `GET /results/{result_id}/top_cams`
#### `GET /results/{result_id}/confident_results`
#### `GET /results/{result_id}/scan_file`

Returns a single field from a result document. Returns `404` if the result
or the field does not exist.

---

#### `GET /results/{result_id}/{field}`

Generic field getter — returns any named field from a result document.

---

### Chat Logs

#### `POST /chatlogs`

Upserts a chat log document matched by `session_id`. On first call it inserts
with `created_at`; on subsequent calls it updates `updated_at` and merges
`result_ids` using set semantics (no duplicates).

**Request body (JSON):**
```json
{
  "session_id": "abc-123",
  "patient_id": "P-001",
  "messages": [
    { "role": "user", "content": "Patient reports chest pain." },
    { "role": "assistant", "content": "Based on the X-ray..." }
  ],
  "result_ids": ["<ObjectId>", "<ObjectId>"]
}
```

**Response:**
```json
{ "status": "ok" }
```

---

#### `GET /chatlogs/{session_id}`

Returns a full chat log document by `session_id`.

---

#### `GET /chatlogs/{session_id}/messages`

Returns only the `messages` array for a given session.

---

#### `GET /chatlogs/by-patient/{patient_id}`

Returns all chat logs for a patient, sorted by `created_at` descending.
Returns up to 100 results. Each document includes `session_id`, `messages`,
`result_ids`, and `created_at`.

---

### Health

#### `GET /health`

Returns `{"status": "ok"}`. Used by the Docker healthcheck.

## Collections Reference

### `analysis_results`

| Field | Type | Description |
|---|---|---|
| `_id` | ObjectId | Auto-generated by MongoDB |
| `created_at` | datetime | Set on insert by this service |
| `pathologies` | object | All pathology scores from DenseNet121 |
| `confident_results` | object | Pathologies scoring above 0.5 |
| `top_cams` | object | 224x224 CAM arrays for top pathologies |
| `primary_embedding` | array | Mean spatial feature vector |
| `scan_file` | string | Saved PNG filename in the scan volume |
| `source_filename` | string | Original uploaded filename |

### `chat_logs`

| Field | Type | Description |
|---|---|---|
| `_id` | ObjectId | Auto-generated by MongoDB |
| `session_id` | string | Unique session identifier (upsert key) |
| `patient_id` | string | Associated patient ID |
| `messages` | array | `{role, content}` objects |
| `result_ids` | array | Linked analysis result ObjectIds |
| `created_at` | datetime | Set on first insert only |
| `updated_at` | datetime | Updated on every save |

## Environment Variables

| Variable | Default | Description |
|---|---|---|
| `MONGO_URL` | `mongodb://localhost:27017` | MongoDB connection string |
| `DB_NAME` | `xray_service` | Database name |
| `COLLECTION_NAME` | `analysis_results` | Analysis results collection name |

## Running Locally (via Docker Compose)

The service is configured in `docker-compose.yml` as `db-service` on port **8002**.
It depends on the `mongo` container being healthy before starting.

```bash
docker compose build db-service
docker compose up db-service
```

Test with:
```bash
# Health check
curl http://localhost:8002/health

# Save a result
curl -X POST http://localhost:8002/results \
  -H "Content-Type: application/json" \
  -d '{"pathologies": {"Cardiomegaly": 0.87}, "scan_file": "test.png"}'

# Save a chat log
curl -X POST http://localhost:8002/chatlogs \
  -H "Content-Type: application/json" \
  -d '{"session_id": "test-123", "patient_id": "P-001", "messages": []}'
```
